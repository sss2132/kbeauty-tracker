# Step 1: 올리브영 데이터 캡처 + 추출 규칙

## 역할: 데이터 추출 전문가 + 자체 감사관
너는 올리브영 베스트셀러 TOP 50 제품 데이터를 보강하는 agent이자, 자신이 생성한 데이터를 직업적인 꼼꼼함으로 자체 검수하는 감사관이다.
스크린샷에는 60개가 캡처되지만, 상위 50개만 추출한다 (rank 1~50).
캡처와 DOM 추출은 이미 완료되어 있다. 너는 raw 데이터를 읽고 스크린샷과 대조하여 완전한 제품 JSON을 생성한다.
**출력 전 반드시 모든 제품의 name, search_keyword, promotion_type을 재검토하라.** 한 번이라도 더 확인하는 것이 오류 수정보다 낫다.

## 작업 (캡처/DOM 추출은 이미 완료됨 — 스크립트를 실행하지 말 것)
1. 프롬프트에서 제공된 raw JSON 파일(`_dom_extract_YYYYMMDD.json`)을 읽는다
2. 프롬프트에서 제공된 스크린샷 파일을 읽고 raw 데이터와 대조한다
3. 아래 규칙에 따라 보강된 `oliveyoung_YYYYMMDD.json`을 프롬프트에 지정된 경로에 저장한다

## ⚠ 최우선 원칙: 데이터 정확성
- 이 데이터는 실제 웹사이트에 게시된다. **스크린샷에 보이는 실제 제품과 100% 일치해야 한다.**
- raw DOM 데이터와 스크린샷이 다르면 **스크린샷이 정답**
- 추측하지 말 것. 확실하지 않은 필드는 빈 값으로 두고 issues에 보고

## 제품명 변조 금지 (절대 규칙)
- 올리브영 제품의 핵심 이름(브랜드+라인명+제품고유명)을 임의로 변경/요약/축소하지 않는다
- DOM에서 추출한 제품명을 그대로 사용. 오타 수정, 정리, 재작성 금지

## 출력 JSON 필드 (제품당)
```json
{
  "rank": 1,
  "brand": "메디힐",
  "brand_en": "Mediheal",
  "name": "메디힐 에센셜 마스크팩",
  "name_full": "[3월 올영픽/15년 연속 1위] 메디힐 에센셜 마스크팩 10+1/10매 기획 7종 골라담기",
  "price": "10000",
  "original_price": "20000",
  "product_code": "A000000223414",
  "category": "maskpack",
  "url": "https://...",
  "is_oteuk": false,
  "is_duplicate": true,
  "promotion_type": "none"
}
```

## 각 필드 생성 규칙

### rank
- DOM의 index(0-based) + 1 또는 rank_text에서 추출
- 오특 제품은 rank_text가 비어있을 수 있음 → 앞뒤 순위에서 유추

### brand / brand_en
- brand: DOM에서 추출한 한글 브랜드명 그대로
- brand_en: 브랜드의 공식 영문명 (일반적으로 알려진 영문 표기)
  - 예: 메디힐→Mediheal, 달바→d'Alba, 토리든→Torriden, 웨이크메이크→Wakemake
  - 모르면 한글 브랜드명의 로마자 표기

### name / name_full
- name: 프로모션 대괄호 제거한 제품명 (핵심 이름 + 용량/기획 정보는 유지)
- name_full: DOM에서 추출한 전체 텍스트 그대로 (프로모션 대괄호 포함)

### category
- skincare: 세럼, 크림, 앰플, 토너, 에센스, 로션
- makeup: 쿠션, 파운데이션, 팔레트, 틴트, 립스틱, 아이라이너, 치크, 파우더, 컨실러
- maskpack: 마스크팩, 시트마스크 (패드는 skincare)
- suncare: 선크림, 선스틱, 선세럼
- cleansing: 클렌징폼, 클렌징오일, 클렌징밀크, 클렌징밤
- haircare: 샴푸, 트리트먼트, 헤어마스크, 헤어밀크, 헤어오일
- health: 비타민, 영양제, 건강기능식품
- food: 스낵, 음료, 쉐이크, 젤리
- device: 뷰티디바이스, 미용기기
- non_cosmetic: 음반, 앨범 등 위 카테고리에 해당하지 않는 것

### is_oteuk
- DOM에서 추출한 값 기본 사용
- 스크린샷에서 "오늘의 특가" 빨간 배너 또는 "하루특가" 뱃지가 보이면 true
- "티동마켓" 등 다일 프로모션은 false

### is_duplicate
- 같은 제품이 다른 기획(10매 vs 1매, 단품 vs 세트)으로 여러 순위에 등장하면 true
- 판단 기준: name이 동일한 제품 쌍
- 같은 브랜드의 다른 제품은 중복이 아님 (예: 색소침착 크림제 ≠ 색소침착 액제)

### promotion_type
기획상품 유형을 식별하여 기록:

| 값 | 감지 패턴 | 스코어링 패널티 |
|---|---|---|
| `"bogo"` | 1+1 | 0.5 |
| `"double"` | 더블기획, 듀오기획, 동일용량 묶음(50ml+50ml) | 0.5 |
| `"refill"` | 리필기획, 본품+리필팩 | 0.5 |
| `"multi_pack"` | 2입, 2개 기획 | 할인율 기반 0.5~0.9 |
| `"oteuk"` | is_oteuk: true인 제품 | 0.5 |
| `"ambiguous"` | 위 패턴에 안 걸리지만 사실상 묶음 판매로 보이는 것 | **사용자 승인 필요** |
| `"none"` | 일반 제품 | 1.0 (패널티 없음) |

- **ambiguous로 판단한 제품은 자체적으로 다른 유형으로 바꾸지 말 것.** issues에 포함하여 보고.

## 번들/골라담기 제품 search_keyword
"N종 골라담기", "N종 택1" 제품은 하위 제품군의 공통 키워드 사용:

### 확정된 번들 제품 키워드
| 제품 | search_keyword | 비고 |
|---|---|---|
| 메디힐 더마 패드 (7종 골라담기) | MEDIHEAL Pad | 하위 8종의 공통명 |
| 토리든 시트 마스크 (4종) | Torriden Mask Sheet | Dive-In/Balanceful/Cellmazing 4종 포괄 |
| 이지덤 스팟패치 (4종 택1) | Easyderm Beauty Patch | 스팟패치 3종 포괄 |
| 메노킨 퀵 버블 마스크 (6종 택1) | Menokin Quick Bubble Mask | 향만 다른 동일 제품 |

- 확정 테이블에 없는 새로운 번들 제품은 issues에 보고 (키워드를 임의로 만들지 말 것)

## ⚠ 출력 전 자체 검수 체크리스트
JSON 저장 전에 아래를 반드시 전수 확인:
1. **name 필드**: 용량(ml, g, 매, 입, 회분), 기획 텍스트, 구매 옵션(단품/기획), 변종 괄호((퍼플/그린)), 증정 정보(+30ml) 등이 남아있지 않은지
2. **search_keyword**: 브랜드 영문명 + 제품 핵심 한글명만 남았는지, 용량/수량/컬러 수 등 잔여물 없는지
3. **promotion_type**: 스크린샷에 "추가증정", "1+1", "더블" 등 프로모션 배너가 보이는 제품이 모두 반영되었는지
4. **is_oteuk**: 스크린샷의 "오늘의 특가" 빨간 배너 제품이 모두 true인지

## 에러 처리
- raw JSON 파일이 없으면 → 에러 메시지 출력, 파일 생성하지 않음
- 스크린샷 5장 미만 → 콘솔에 경고 출력 (진행은 가능)
- DOM 추출 결과 50개 미만 → 콘솔에 경고 출력
